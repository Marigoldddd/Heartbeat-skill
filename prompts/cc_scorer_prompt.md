# CC 语义评分 Prompt — 好感度分析

## 你的任务

你将收到一段按时间窗口分组的聊天记录（JSON 格式）。请对每个时间窗口分别评估「我方」和「对方」的好感度，输出结构化评分。

**这是纯语义分析**，不要做机械的关键词统计，而是像一个读过很多恋人对话的人一样，感受字里行间的情绪和态度。

---

## 输入格式

```json
{
  "windows": [
    {
      "window": "2024-W01",
      "label": "2024-01-01",
      "messages": [
        {"sender": "me",   "content": "hi，在干嘛", "ts": "2024-01-01T09:00:00"},
        {"sender": "them", "content": "在等你消息呢！", "ts": "2024-01-01T09:01:00"},
        ...
      ]
    },
    ...
  ]
}
```

---

## 输出格式

必须输出一个合法的 JSON 数组，每个窗口一个对象，格式严格如下：

```json
[
  {
    "window": "2024-W01",
    "me_cc": 78,
    "them_cc": 85,
    "me_reasoning": "主动发起，表达温柔，有主动拉近距离的意愿",
    "them_reasoning": "回应热情，情感词丰富，明显在等待联系",
    "events": ["热恋初期"],
    "confidence": "high"
  }
]
```

字段说明：
- `me_cc` / `them_cc`：0–100 的好感度评分（整数）
- `me_reasoning` / `them_reasoning`：简短的评分依据（15字以内）
- `events`：该窗口重要事件标签（简短，≤6字，可为空数组）
- `confidence`：`"high"` / `"medium"` / `"low"`（消息太少时用 low）

---

## 评分维度（用你的语言理解综合判断）

### 好感度高的信号
- 主动发起话题、找话说、不想结束对话
- 情感表达真诚（不是例行公事）
- 在意对方的状态，主动问候
- 撒娇、亲昵称呼、分享生活细碎
- 回复及时，不让对话冷场
- 对对方说的话有真实回应（不是敷衍）

### 好感度低的信号
- 单字/词回复（嗯 / 哦 / 好 / 随便 / 知道了）
- 打断对话、不接话茬
- 有意无意地冷漠（不问、不回应、不关心）
- 抱怨、指责、消极情绪主导
- 长时间不回——即使文字上没有表现
- 「随便」「算了」「不用了」等放弃信号

### 模糊信号的处理
- "嗯" 在热恋期 = 含羞，在衰退期 = 敷衍 → 结合上下文判断
- 抱怨也可以是高好感（说明在乎、愿意沟通）→ 看语气
- 沉默/不回复本身也是信号，但只在消息时间间隔异常时才纳入

## 评分参考档位

| 分值 | 描述 |
|------|------|
| 85–100 | 明显热情投入，积极主动，情感浓烈 |
| 70–84 | 好感明确，态度积极，愿意付出时间 |
| 55–69 | 关系正常维系，没有特别热情也没有冷淡 |
| 40–54 | 开始有距离感，回应减少，被动居多 |
| 25–39 | 明显冷淡，敷衍，或有负面情绪主导 |
| 0–24 | 几乎没有好感，冷战/放弃/断联状态 |

---

## 重要提示

1. **只输出 JSON**，不要有任何解释文字、markdown 格式或代码块包裹
2. 每个窗口的 `window` 字段值必须与输入完全一致
3. 消息少于 3 条的窗口，`confidence` 设为 `"low"`，评分向 50 偏移
4. 如果某方在该窗口完全没有消息，对应分数设为 `null`
5. events 标签参考：`热恋高峰` / `开始冷战` / `和好回暖` / `情感骤降` / `疏离加剧` / `主动修复` / `单方付出`
